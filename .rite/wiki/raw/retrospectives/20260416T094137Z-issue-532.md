### Summary

Wiki 機能（経験則の自動蓄積）が実ワークフロー経路から一度も発火していなかった根本原因を修正した親 Issue。6 件の子 Issue に分解して対応:

1. **Phase 0 (#533)**: 実ワークフロー trace で盲点を計測し、3つの決定的な実装欠陥を特定
2. **Phase 1-A (#534)**: start.md Phase 5.7.2 が `gh issue close` を直接実行していたのを `/rite:issue:close` Skill 呼び出しに置換（close.md Phase 4.4.W.2 の 100% silent skip を解消）
3. **Phase 1-B (#535)**: review.md / fix.md の W Phase 到達経路を確保（early return による skip を修正）
4. **Phase 1-C (#536)**: wiki-growth-check.sh に PR↔raw 対応検知を追加（regression 早期検出）
5. **Phase 2-A (#537)**: pr/cleanup.md に /rite:wiki:ingest 自動発火 Phase を追加
6. **Phase 3 (#538)**: 監視 blind spot の最終封鎖 + SKILL.md trouble-shooting

### Key Learnings

- **「直ったと主張する fix は、shell script 単体動作ではなく、実ワークフロー経路の commit 履歴で発火を実証しなければならない」** — #528/#529 で手動テストでは動作確認できたが、自然な workflow 経路からの発火はゼロ件だった
- 親 Issue → 子 Issue の分解と並行実装が効果的だった（6 件の子 Issue を計画的に実行）
- silent skip の問題は多層防御（Phase 実行保証 + 監視 + trouble-shooting ガイド）で封鎖する必要がある
