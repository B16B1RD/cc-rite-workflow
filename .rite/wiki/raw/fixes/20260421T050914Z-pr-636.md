### Fix Patterns

- **Silent-false-pass detection via 3-axis assertion**: stderr suppress + exit code 未検査 + state transition 未検証 の 3 条件が揃った subshell test は「スクリプト未実行でも PASS」する silent-failure path を形成する。pre-state に target value を埋め込んで事後条件のみで assert する test は error injection で verify すべし (TC-634-K/L の silent-false-pass パターン)。
- **Fault injection via PATH override**: mv 失敗 / jq write 失敗 など shell builtin やコマンド失敗を test 再現するには、fake binary を作成して PATH を prepend する方式が portable で root 不要。chmod -w / chattr +i よりも確実 (TC-634-M/N で stop-guard.sh の error_count_mv_failed diag log と flow-state-update.sh の mv failed stderr を verify)。
- **set -e + subshell exit-code 捕捉**: `set -euo pipefail` 下で subshell が非 0 exit する test では `( ... ) || rc=$?` パターンで rc を capture しないと親 script が即座に kill される。事前に `rc=0` で初期化してから `|| rc=$?` を付ける。
- **Line-number reference anti-pattern**: コメント内の `file.sh L247-250` 形式の行番号参照は refactor で silent drift する。semantic anchor (例: "error_count atomic write 後 mv 失敗 path") + `(line-number 参照を避ける理由は cycle 8 F-05 参照)` trailer に置換する convention がこのリポジトリで確立されている。
