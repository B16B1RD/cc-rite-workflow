### Fix Patterns

Stub 残置 formatting pattern の事前確認 — PR シリーズで複数 references を抽出する場合、最初の PR で確立された stub 残置 markdown formatting (`> **Moved (Issue #N PR M/T)**:` 形式) を新規 reference 抽出時にも踏襲することで、保守者が `git grep` で stub 一覧を機械検索できる traceability を維持する。新規 reference で独自フォーマットを採用すると、PR 番号 trace 不揃いを cosmetic 指摘で再修正することになる。

「Partial Moved」prefix の有用性 — critical 警告を本体に残す部分残置経路では、`> **Source of Truth**:` よりも `> **Partial Moved (...)**:` + `critical 警告は本体に維持` の組み合わせがより明示的。完全移管 (Full Moved) と部分残置 (Partial Moved) の semantic 区別が読者に伝わる。
