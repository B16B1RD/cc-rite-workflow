### Fix Patterns
本 fix は cross-boundary state transfer の canonical pattern を Phase 5.5.2 metrics block に適用した。
旧実装は plan_deviation_count を success 経路で stdout/stderr に emit せず、Bash tool 境界でシェル変数が
消失するため、Claude が下流 heredoc placeholder に literal substitute できない構造的バグだった。
ブロック内コメントは partial corruption 防止を主張していたが、stdout emit 欠落で目的を達成できていなかった
(self-contradiction)。修正: if 文を if-else に拡張し、success 経路で `[CONTEXT] PLAN_DEVIATION_COUNT=$N`
を stderr に emit。METRICS_SKIPPED と排他になるよう設計。

### Lesson learned
- bash block 内でシェル変数として計算した値を、別の Bash tool invocation の heredoc に literal substitute する場合、
  stdout/stderr に明示的に emit する必要がある (シェル変数は invocation 境界で消失するため)
- block 内コメントが「○○ 防止」を主張していても、対称な emit が success/failure 両経路に
  実装されていない場合は self-contradiction として review reviewer 主指摘の対象になる
