### Fix Patterns

- **`grep -c || echo 0` 罠の修正**: bash の `grep -c` は match 0 件でも stdout に "0\n" を出力する仕様 ([POSIX grep(1p)](https://pubs.opengroup.org/onlinepubs/9699919799/utilities/grep.html))。`|| echo 0` fallback と組み合わせると count が "0\n0" の改行付き 2 行に汚染され、後続の integer test が exit 2 で stderr 警告を吐く。リポジトリ慣用は `|| true` + `${var:-0}` パターン (session-start.test.sh:760 に作者警告コメント有り)。**新規 bash test 追加時は必ず既存 metatest を参照して同パターンを使うべし。**

- **summary 行 self-reference の自動同期**: bash test の summary 行で実ファイル名を hardcode するとファイル rename 時に drift する。`$(basename "$0")` で自動同期させると rename しても整合性が保たれる。
