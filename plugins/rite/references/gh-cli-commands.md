## Issue Operations

### Fetch Issue Details

```bash
# Basic information
gh issue view {issue_number} --json number,title,body,state,labels,milestone,projectItems

# All fields
gh issue view {issue_number} --json number,title,body,state,labels,milestone,projectItems,assignees,author,createdAt,updatedAt
```

### List Issues

```bash
# List OPEN Issues
gh issue list --state open --json number,title,state

# Filter by label
gh issue list --label "bug" --state open

# Filter by assignee
gh issue list --assignee "@me" --state open
```

### Create Issue

```bash
# ✅ SAFE: --body-file pattern (see "Safe Issue/PR Body Updates" for details)
# Note: Fixed string HEREDOC guarantees non-empty content at write time, so empty check (defense layer 2) is omitted. Always add empty check for dynamic generation.
tmpfile=$(mktemp)
trap 'rm -f "$tmpfile"' EXIT

cat <<'BODY_EOF' > "$tmpfile"
調査内容をここに記載
BODY_EOF

gh issue create \
  --title "{title}" \
  --body-file "$tmpfile" \
  --label "bug"
```

### Update Issue

```bash
# ✅ SAFE: --body-file pattern (always follow the "Safe Issue/PR Body Updates" section procedure)
gh issue edit {issue_number} --body-file "$tmpfile"  # $tmpfile is the path generated by mktemp

# Add label
gh issue edit {issue_number} --add-label "{label}"

# Set assignee
gh issue edit {issue_number} --add-assignee "@me"
```

### Issue Comments

```bash
# ✅ OK: --body for short fixed strings (no special characters)
gh issue comment {issue_number} --body "{comment}"

# ✅ SAFE: --body-file for long or special character comments
tmpfile=$(mktemp)
trap 'rm -f "$tmpfile"' EXIT
cat <<'EOF' > "$tmpfile"
コメント本文
EOF
gh issue comment {issue_number} --body-file "$tmpfile"

# List comments
gh api repos/{owner}/{repo}/issues/{issue_number}/comments --jq '.[].body'
```

### Close Issue

```bash
# Close with comment (--comment only accepts short fixed strings)
gh issue close {issue_number} --comment "{reason}"

# Reopen
gh issue reopen {issue_number}
```

## PR Operations

### Fetch PR Details

```bash
# Basic information
gh pr view {pr_number} --json number,title,body,state,isDraft,headRefName

# Including review status
gh pr view {pr_number} --json number,title,state,reviews,reviewRequests
```

### List PRs

```bash
# My PRs
gh pr list --author "@me" --state open

# Awaiting review
gh pr list --search "review:required"

# Drafts
gh pr list --draft
```

### Create PR

> **⚠️ IMPORTANT**: Before creating a PR, verify that commits exist and branch is pushed. See [gh-cli-patterns.md - Category 5](./gh-cli-patterns.md#category-5-graphql-pr-creation-error--10-sessions) and [graphql-helpers.md - Combined Guard](./graphql-helpers.md#combined-guard-recommended).

```bash
# ✅ SAFE: --body-file pattern
# Note: Fixed string HEREDOC guarantees non-empty content at write time, so empty check (defense layer 2) is omitted. Always add empty check for dynamic generation.
tmpfile=$(mktemp)
trap 'rm -f "$tmpfile"' EXIT

cat <<'BODY_EOF' > "$tmpfile"
初期実装
BODY_EOF

gh pr create \
  --title "{title}" \
  --body-file "$tmpfile" \
  --base "{base_branch}" \
  --draft
```

### Update PR

```bash
# Remove draft status
gh pr ready {pr_number}

# ✅ SAFE: --body-file pattern (always follow the "Safe Issue/PR Body Updates" section procedure)
gh pr edit {pr_number} --body-file "$tmpfile"  # $tmpfile is the path generated by mktemp

# Add reviewer
gh pr edit {pr_number} --add-reviewer "{username}"
```

### Get PR Diff

```bash
# Full diff
gh pr diff {pr_number}

# File list only
gh pr diff {pr_number} --name-only

# 🚫 PROHIBITED: gh pr diff does NOT support -- <files> for per-file filtering
gh pr diff {pr_number} -- path/to/file.md
# → Silently fails or returns unexpected results

# 🚫 PROHIBITED: gh pr diff does NOT support --stat
gh pr diff {pr_number} --stat
# → Error: unknown flag: --stat
# → Use gh pr view --json files to get per-file additions/deletions instead

# ✅ SAFE: Filter diff sections by file pattern using awk
# Resets found=0 on each new diff header, sets found=1 on pattern match
# Note: Do NOT use \! for negation in awk — it is invalid awk syntax,
#       and bash may also interpret ! specially (see "Shell Escaping Notes" below)
gh pr diff {pr_number} | awk '
  /^diff --git/ { found=0 }
  /^diff --git.*target_file_pattern/ { found=1 }
  found { print }
'
```

## Projects Operations

### List Project Fields

```bash
gh project field-list {project_number} --owner {owner} --format json
```

### Add Item

```bash
gh project item-add {project_number} --owner {owner} --url {issue_url}
```

### Update Field

```bash
# Update Status (Single Select)
gh project item-edit \
  --project-id {project_id} \
  --id {item_id} \
  --field-id {field_id} \
  --single-select-option-id {option_id}

# Update text field
gh project item-edit \
  --project-id {project_id} \
  --id {item_id} \
  --field-id {field_id} \
  --text "{value}"
```

## Repository Information

### Owner and Repository Name

```bash
# Fetch both at once
gh repo view --json owner,name --jq '{owner: .owner.login, repo: .name}'

# Fetch individually
gh repo view --json owner --jq '.owner.login'
gh repo view --json name --jq '.name'
```

### Default Branch

```bash
gh repo view --json defaultBranchRef --jq '.defaultBranchRef.name'
```

## Branches and Commits

### Check Branch

> **DO NOT** use exit code (`&&`, `||`, `$?`) to determine branch existence. `git branch --list` always returns exit code 0 regardless of whether a match is found. Check the **output** instead (non-empty = exists).

```bash
# Current branch
git branch --show-current

# Check local branch existence (check OUTPUT, not exit code)
local_match=$(git branch --list "{branch_name}")
if [ -n "$local_match" ]; then
  echo "BRANCH_EXISTS"
else
  echo "BRANCH_NOT_FOUND"
fi

# Check remote branch existence (check OUTPUT, not exit code)
remote_match=$(git branch -r --list "origin/{branch_name}")
if [ -n "$remote_match" ]; then
  echo "REMOTE_EXISTS"
else
  echo "REMOTE_NOT_FOUND"
fi
```

### Push

```bash
# Push new branch
git push -u origin {branch_name}

# Normal push
git push
```

### Commit

```bash
# Conventional Commits format
# Note: Co-Authored-By is the standard format when using Claude Code
#       The model name (currently: Claude Opus 4.6) may change in the future
#       Modify as appropriate for other collaborators
git commit -m "$(cat <<'EOF'
feat(scope): add new feature

Description of the change.

Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
EOF
)"
```

## Useful Patterns

### Existence Check

```bash
# Check if PR exists
gh pr list --head {branch_name} --json number --jq '.[0].number // empty'

# Check if Issue is registered in Project
gh api graphql -f query='
query($owner: String!, $repo: String!, $number: Int!) {
  repository(owner: $owner, name: $repo) {
    issue(number: $number) {
      projectItems(first: 1) {
        totalCount
      }
    }
  }
}' -f owner="{owner}" -f repo="{repo}" -F number={issue_number} --jq '.data.repository.issue.projectItems.totalCount'
```

### Conditional Branching

```bash
# Branch based on whether PR is draft
IS_DRAFT=$(gh pr view {pr_number} --json isDraft --jq '.isDraft')
if [ "$IS_DRAFT" = "true" ]; then
  echo "Draft PR"
else
  echo "Ready for review"
fi
```
